Inside Airbnb Dashboard Redesign

Redesign and optimization of the Inside Airbnb dashboard to enhance user experience and data exploration capabilities

ðŸ¤” Why This Project?

Problem Statement

The current Inside Airbnb dashboard, which visualizes Airbnb property data for specific regions, faces several design and functionality challenges that hinder user experience and data exploration.

  • Overuse of Vertical Space: The interactive map dominates the dashboard, occupying excessive vertical space, forcing other statistical sections into a vertical arrangement, violating principles of effective data comparison
  • Limited Filtering Options: The dashboard restricts data exploration to a single country at a time, limiting its utility for users seeking granular insights across states or cities
  • Weak Readability: Overly detailed descriptions and repetitive percentage breakdowns increase cognitive load and complicate data scanning, reducing clarity and user engagement

Objectives & Goals

  • Redesign the dashboard layout to optimize space usage, reducing the map's dominance and enabling side-by-side data comparisons
  • Expand filtering options to include state- and city-level data, improving user exploration capabilities
  • Enhance readability by simplifying descriptions, embedding them in tooltips, and reducing redundancy
  • Create a more intuitive, accessible, and functional dashboard for stakeholders in the short-term rental market
  • Implement interactive elements to improve user engagement and reduce cognitive load

ðŸ“Š Data Description

Dataset Overview

The dashboard leverages data sourced from the official Inside Airbnb website, focusing on Airbnb listings in three major Australian cities: Sydney, Melbourne, and Brisbane.

  • Host IDs: Unique identifier for hosts, used to track listings per host
  • Room Type: Categorical data describing property type (entire home, private room)
  • Average Nights Booked: Numerical data indicating average booking duration per listing
  • Price per Night: Numerical data representing cost per night for each listing
  • Average Income: Numerical estimates of income generated by each listing
  • Licenses: Categorical details of licensing information for properties
  • Number of Short-Term Rentals: Numerical counts of total short-term rental listings

Data Preparation & Quality

  • Data extracted from Inside Airbnb (https://insideairbnb.com/sydney) as of September 2024
  • Cleaned missing or incomplete fields to ensure data integrity
  • Standardized data types (numerical for price and income, categorical for room type and licenses)
  • Aggregated data by city for consistent analysis across regions
  • Limited to three cities and September 2024 data, restricting multi-state comparisons

Analytics Approach

The dashboard was developed through a structured process to address the identified challenges:

  • Exploratory Data Analysis (EDA): Analyzed relationships between variables (price per night, average income, room type) to identify patterns and correlations
  • Layout Redesign: Reduced the map's size to balance space allocation and enable side-by-side comparisons of statistical sections
  • Feature Engineering: Created aggregated metrics (average price per city, booking trends) to support filtering and visualization
  • Interactive Elements: Implemented tooltips for descriptions and interactive graphs to enhance user engagement and reduce redundancy
  • Testing and Validation: Conducted extensive testing to ensure callback functions and interactive components worked seamlessly
